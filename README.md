# Applied AI for Cybersecurity

This repository contains the course website and teaching materials for the undergraduate module:

**Applied AI for Cybersecurity**

The module is designed for **Level 6 students** in a UK university context and introduces the practical use of artificial intelligence in cybersecurity. It covers both **AI for cybersecurity** and the **security of AI systems**, with emphasis on critical thinking, trustworthy deployment, and professional judgement.

---

## Module focus

The course explores how AI can support cybersecurity tasks such as:

- intrusion detection
- phishing and malicious URL analysis
- alert triage and security analytics
- malware-related analysis
- anomaly detection
- analyst support with generative AI and LLMs

It also examines the risks and limitations of AI-enabled cyber systems, including:

- adversarial machine learning
- evasion and poisoning attacks
- model misuse and overreliance
- prompt injection and insecure output handling
- privacy, explainability, robustness, and governance

The overall aim is not only to teach students how to apply AI techniques, but also how to evaluate whether such systems are suitable for real deployment.

## Course structure

The module is organised into five teaching weeks:

1. **Foundations of Applied AI for Cybersecurity**
2. **Data, Features, and Classical Machine Learning for Security Analytics**
3. **Deep Learning and Generative AI in Cybersecurity**
4. **Attacking and Defending AI Systems**
5. **Trustworthy Deployment, Governance, and Capstone Case Study**

In addition to weekly lecture pages, the site includes:

- a syllabus
- a weekly schedule
- a labs page
- an assessment page
- a curated references page

---

## Educational approach

This module is designed as an **applied**, **critical**, and **professionally relevant** course.

It emphasises:

- realistic cybersecurity workflows
- practical data analysis
- careful evaluation of AI outputs
- awareness of operational limitations
- secure and responsible use of AI tools
- clear communication of technical judgement

The module does **not** treat AI as automatically reliable or automatically beneficial. Students are encouraged to ask whether an AI-based approach is justified, how it should be evaluated, what risks it introduces, and whether it should actually be trusted in practice.
